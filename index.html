  • Cryptography:

    Cryptography is a modern technique to protect digital data. These techniques are derived from a predetermined set of rules which transforms the message of the trusted user. This message cannot be recognized by the unauthorized user.

    Symmetric-key cryptography:

    When a secret key is used for both encryption and decryption operations, the process is known as symmetric key cryptography, also referred to as symmetric encryption. Some of the most common use cases for symmetric cryptography include:

    • Banking:

      Encrypting credit card information or other personally identifiable information (PII) required for transactions.

    • Data Storage:

      Encrypting data stored on a device when that data is not being transferred.

  • Encryption & Decryption:
    • Encryption:

      It is a cryptographic technique using which the sender's message is converted into a secret code that cannot be recognized without a security key. Such type of encrypted data is called cipher text.

    • Decryption:

      The process of conversion of the cipher text to the plain text is called decryption. To decrypt the encrypted data, the user needs a key. The access of this key is given only to the receiver(the trusted users).

  • Caesar Cipher:

    The Caesar Cipher is one of the earliest and most straightforward encryption methods. It's basically a substitution cipher, in which each letter of a given text is substituted with a letter located a certain number of positions down the alphabet. With a shift of one, for example, A would be replaced by B, B by C, and so on. Julius Caesar is said to have called the approach after himself, as he used it to communicate with his officials.

    To cipher a given text, we need an integer value called shift, which tells how far down each letter of the text has been relocated.

    The encryption can be represented using modular arithmetic by converting the letters to numbers using the A = 0, B = 1,..., Z = 25 approach.

  • Monoalphabetic:

    A monoalphabetic cypher, often known as a basic substitution cypher, has a predefined substitution scheme. That is, for each letter of the alphabet, the substitution is fixed. If "a" is encrypted to "R," we replace the letter "a" in the plaintext with the letter "R" in the ciphertext every time we encounter it.

    "Hello World" becomes "Htssg Wgksr" for example, because each letter is encoded as the next letter in the alphabet. It's easier to construct the ciphertext alphabet first, then encrypt by comparing it to the plaintext alphabet when executing a simple substitution manually.

  • Rail-Fence:

    To use the Rail Fence Cipher to encrypt a message, write your message in zigzag lines across the page, then read off each row. To begin, you'll need a key, which for this cipher is the number of rows you'll have. Then you start writing the plaintext letters diagonally down to the right until you reach the key's chosen number of rows. Then you bounce up diagonally till you reach the first row once more. This continues till the plaintext is finished.

    The Rail Fence Cipher decryption technique entails rebuilding the diagonal grid that was used to encrypt the message. We begin writing the message, but leave a dash in the places that have yet to be filled. Replace all the dashes with the corresponding letters and read the plaintext from the table gradually.

    To begin, we create a grid with as many rows as the key and as many columns as the ciphertext's length. The initial letter is then placed in the top left square, with dashes diagonally downwards indicating where the characters will be. We return to the top row and insert the next letter into the ciphertext. Continue in this manner across the row, and when you reach the end, begin the next row.
